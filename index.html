결제선생 신규 서비스 개발 및 유지보수

2021.08 - 현재

사업장에서 고객에게 비대면 결제를 요청할 수 있도록 청구서 발송, 발송 예약, 발송 관리, 수납 관리, 매출 보고서, 기프티콘 선물하기 등 결제 및 매출 관리를 포괄하는 웹페이지 및 웹뷰를 개발하였습니다.

  • 기존 서비스 유지보수 과정에서 중복된 로직을 모듈화하고, 유연한 UI 컴포넌트 설계를 도입하여 여러 컴포넌트를 합성해 새로운 컴포넌트를 만들 수 있도록 개선했습니다. 이를 통해 재사용성을 높이고, 유지보수와 확장이 용이한 구조를 구축했습니다.
  • 주도적으로 동료들과 함께 코드 컨벤션과 커밋 컨벤션을 지정하여 일관된 코딩 스타일과 명확한 커밋 메시지의 중요성을 전파했습니다. 또한, 개발 일정을 체계적으로 관리하여 팀의 생산성과 프로젝트의 일정 준수를 향상시켰습니다.
  • 웹팩 빌드 설정을 수정하여 초기 로드해야 할 벤더(vendor) 사이즈를 줄이고, page router 파일 및 컴포넌트 import 시 코드 스플리팅(code splitting)과 트리 쉐이킹(tree shaking)을 적용하여 필요한 모듈만 로드하도록 개선했습니다. 이를 통해 초기 렌더링 속도를 크게 향상시켰습니다. 또한, 대역폭이 낮은 개발 환경에서 최적화를 위해 Node.js로 프로젝트를 실행할 때 compression 기능을 추가하여 렌더링 속도를 개선했습니다.
  • Webpack3에서 Webpack4로 버전 업그레이드를 통해, 빌드 시 필요한 플러그인을 패키지에 다운로드 받아 사용하던 환경에서 Webpack4의 내장된 메서드를 활용하여 빌드 속도를 향상시켰습니다.
  • Axios 인터셉터를 활용하여 토큰 기반 API 호출 시 요청 헤더에 토큰 값을 자동으로 추가했습니다. 또한, 응답 코드에서 토큰 만료 코드가 수신될 경우, 팝업을 노출시켜 재로그인을 유도하는 기능을 구현했습니다.
  • 사업장 가입 시 정해진 플로우와 뒤로 가기 기능을 제어하기 위해, 내부적으로 히스토리 관리를 구현하여 지정된 흐름에 따라 사용자 이동을 제어할 수 있도록 작업했습니다.
  • 웹뷰와 네이티브 앱 간의 통신을 위한 웹뷰 브릿지를 구현하고, 네이티브 앱 개발자들과 협의하여 여러 개의 브릿지를 하나로 통합했습니다. 이를 통해 통신 효율성을 개선하고 유지보수성을 높였으며, 작업 내용을 문서화하여 체계적으로 정리하였습니다.
  • 프로모션 배너가 추가될 때마다 프로젝트에 이미지 파일을 직접 추가하고 새로 빌드하여 배포하던 기존 방식을 개선했습니다. Firebase를 활용하여 이미지를 원격으로 업로드하고, 원격 구성 기능을 통해 프로모션의 노출 기간, 이미지 URL, 클릭 이벤트 함수명을 관리할 수 있도록 했습니다. 이를 통해 JSON 파일을 읽어 배너를 동적으로 노출함으로써 운영 배포의 빈도를 크게 줄였습니다.
  • 라우트 이동 시 Google Analytics 트리거 이벤트를 설정하여 사용자가 자주 방문하는 페이지를 집계하고 분석하는 데 기여했습니다.
  • 전역 상태 관리 라이브러리인 Vuex를 사용하여 주제별로 파일을 나누어 관리하고, 유지보수가 용이하도록 구조를 개선했습니다. 이를 통해 코드의 가독성을 높이고, 모듈화된 관리로 인해 개발 효율성을 향상시켰습니다.
  • 사업장의 매출과 청구서 수납율에 대한 데이터를 Google Charts의 선 그래프(line chart)와 막대 그래프(bar chart)로 시각화하여 사용자의 데이터를 효과적으로 분석할 수 있도록 했습니다.
  • 프로젝트 진행 중 오픈 소스를 정상적으로 불러오지 못하는 상황에 대비하여, 해당 리소스를 로컬 파일에 추가하고 이를 대체 리소스로 사용할 수 있도록 방어 로직을 구현했습니다. 이를 통해 서비스의 안정성을 높이고, 외부 의존성으로 인한 장애를 최소화했습니다.
  • 세션 스토리지를 간편하게 사용할 수 있는 npm 모듈에서 간헐적인 오류가 발생하여, 내부적으로 Vue 라이브러리를 직접 구현하였습니다.

  • 결제 앱 스킴 호출 후 결제 완료 상태를 확인하기 위해 폴링(polling) 구조를 추가하였습니다.
  • 결제 영수증을 다른 사용자와 공유할 수 있도록 카카오톡 전달, SMS 전송 기능을 추가하고, 이미지 저장을 위해 dom-to-image 라이브러리를 활용했습니다.
  • 공통된 필수 요청 값을 저장하기 위해 전역 상태 관리 라이브러리인 Vuex를 활용하여 작업을 진행했습니다.
